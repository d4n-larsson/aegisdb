Durable memory for AI agents

AegisDB is a small, fast database written in C that gives agents persistent long-term memory — episodic, semantic, and working — over a dead-simple JSON-over-TCP protocol, with vector search, a relationship graph, and crash recovery built in.

What it gives your agents

A memory substrate designed around how agents actually remember — not a generic key-value store bolted on.

🧠

Three memory types

Immutable episodic events, mutable semantic facts, and ephemeral working memory — each with the right lifecycle and guarantees.

🔎

Search that fits recall

Query by semantic vector similarity, by tags, or by time range — backed by purpose-built indexes, not a linear scan.

🕸️

Relationships & traversal

Link memories into a graph and walk it with breadth-first traversal to pull in related context.

💾

Crash-safe persistence

An append-only log with per-record CRC32 checksums; indexes rebuild from the log on startup. Kill it mid-write and nothing is lost.

🔐

Shared & authenticated

Run one server for a whole team: a shared memory pool gated by bearer-token authentication.

⚡

Tiny & dependency-light

A multi-threaded C server with a one-object-per-line protocol any language can speak in a few lines.

One line in, one line out

The wire protocol is newline-delimited JSON over TCP. No SDK required — talk to it with a socket.

Store a memory and search it back

# Insert an episodic memory
{"operation":"insert", "type":"episodic", "tags":["user"], "data":"Prefers dark mode"}
→ {"ok":true, "record":{"id":1, "type":"episodic"}}

# Search by tag
{"operation":"search", "tags":["user"], "top_k":5}
→ {"ok":true, "total":1, "records":[ … ]}

Operations: ping · insert · get · update · delete · search · promote · relate · traverse

Long-term memory for Claude Code

AegisDB ships with a Claude Code integration: an MCP server exposing memory tools, plus hooks that automatically recall relevant memories into context and capture salient ones at session end.

  • Automatic recall on every prompt, within a strict time budget.
  • Automatic capture of what matters, filtered by salience.
  • Per-project namespaces keep memories isolated — or shared, by choice.
  • Graceful degradation: if the backend is down, your session never breaks.

.mcp.json

{
  "mcpServers": {
    "memory": {
      "command": "python3",
      "args": ["-m", "aegis_mcp.server"],
      "env": {
        "AEGIS_NAMESPACE": "my-project",
        "AEGIS_EMBEDDING_DIMENSIONS": "1024"
      }
    }
  }
}

Get started

Clone, build, and run — no external dependencies to install.

Clone & build

git clone https://github.com/d4n-larsson/aegisdb && cd aegisdb && make

Run the server

./build/aegisdb --data-dir ./data --port 9470 --embedding-dim 1024

Talk to it

echo '{"operation":"ping"}' | nc -q1 localhost 9470
